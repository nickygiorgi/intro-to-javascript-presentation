A History of JavaScript
From Wikipedia

Javascript was developed in 10 days in May 1995 by Brendan Eich at Netscape as a lightweight interpreted language for client side scripting within the Netscape web browser
  • It was meant to be a competitor to Microsoft's Visual Basic on Internet Explorer and aimed to appeal to non-professional programmers, to make it easy for everyone to expand and improve contents that could be navigated through Netscape.
  • It was originally developed under the name Mocha and officially called LiveScript when shipped. Only later it was deployed with its current name of JavaScript.
  • In November 1996, Netscape submitted JavaScript to Ecma International and in June 1997 it was accepted and specified by the standard ECMA-262. The standard specification of JavaScript is called ECMAScript, and the latest version is ES 6.

Javascript weirdness
Part 1

JavaScript is a dynamic and (generally) interpreted programming language.
  • dynamic: many operations are performed at runtime instead of compile time. For example, the code in a function can change at runtime.
  • functions are first-class citizens: they are treated like variables, can actually be assigned to variables (will not be global), can be passed to other functions, can be returned to be used later, etc...
  • JavaScript supports closures
  • interpreted: the code is run directly by an interpreter instead of being compiled to machine code (it is translated in subroutines which are already available as machine code)

Javascript weirdness
Part 2

JavaScript is weakly and dynamically typed – it only has numbers and strings but the same variable can hold a number and then a string or viceversa
  • var number = 123;
  • var string = 'abc';
  • you can divide a number by a string! :)

  • you can pass a list of absolutely mixed things to a single method and use it, provided you know how to use it: a fiddle

The DOM

JavaScript is generally used to manipulate the DOM in HTML pages on the web. DOM stands for Document Object Model: a definition and another definition
  • it's a programming interface that models any HTML page on the web as a tree. By navigating the nodes on the tree, it is possible via a scripting language like JavaScript, to change the HTML and modify the page without going back to the server
  • you can use ajax to ask questions to the server without reloading the full page
  • in the document there will always be a body tag
  • a tag can have one unique id and several attributes

How to find elements in the DOM

  • JavaScript allows you to getElementById, getElementByTagName, getElementByClassName
  • Jquery provides css like selectors to get almost everything: a cheatsheet
